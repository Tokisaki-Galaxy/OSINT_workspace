# 美味小土豆：人物关系图谱处理过程与工作笔记

## 1. 任务目标

- 生成一份可读的人物关系图（聚焦“女朋友/关键女性关系”）。
- 记录结论、处理过程、筛选口径与不确定性，便于后续复核。

## 2. 数据来源

本次仅使用仓库内现有数据：

- `美味小土豆/extracted_mds/*.md`（原回答文本与时间元数据）
- `美味小土豆/events_raw.csv`（事件抽取明细）
- `美味小土豆/events.csv`（事件聚合）
- `美味小土豆/社会学侧写.md`（既有结构化结论）

## 3. 检索关键词

用于粗筛关系线索：

- 关系词：`女朋友`、`前女友`、`对象`、`交往`、`cp`、`搭子`
- 目标词：`商会大小姐`、`大小姐`、`JK`
- 时序词：`高中`、`高二`、`大学`、`工作后`

## 4. 处理步骤

1. **全局粗筛**：在 `美味小土豆` 目录内检索关键词，定位高频人物与关系事件。  
2. **证据回看**：优先回看包含明确时间锚点与关系动作的原始 md。  
3. **关系归并**：按“核心人物 / 圈层人物 / 群像人物”分层整理。  
4. **时间线抽取**：仅使用文本内可见锚点（如“高二”“19岁”“工作后”及文件发布时间）。  
5. **可信度标注**：
   - 高：多来源重复 + 事件链完整
   - 中：有明确叙述但边界模糊
   - 低/中低：群像化表达或戏剧化成分高

## 5. 口径约束（重要）

- 本产物是**文本关系图**，不是现实调查结论。  
- 对“JK”采用双口径处理：
  1) 审美/话题标签；
  2) 可能的真实交往对象。  
  二者不混同。
- 若同一人物在不同文本中关系定义冲突，以“保守解释”处理，不强行定性。

## 6. 主要判断依据（摘录）

- “商会大小姐”在多篇文本与事件抽取中重复出现，且覆盖关系形成、共同行动、经济支持等关键环节，因此列为高可信核心人物。
- “浙江有钱大小姐（老板）”具备明确年龄差、互动模式与结束事件（对方结婚），事件完整度较高。
- “CP/后辈/短发眼镜姑娘”处于同一公司场景内，频繁同现，但恋爱边界混杂，故置于中可信层。
- “很多女朋友”结论主要来自“群像叙事 + 快节奏关系 + 自述修辞”，并非稳定一对一关系清单。
